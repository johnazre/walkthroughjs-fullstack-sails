If/Else Statements

If/Else statements are incredibly useful when you're looking to do something specific and nothing else. Think of when you're going through the refrigerator at home for that pastrami sandwich you bought last night and aren't willing to settle for anything less. Nothing else is worth eating. If you find it, you will eat it, else you will just say "whatever" and move on. That's an If/Else statement. My sincerest apologies if you're reading this before lunch. Let's distract you with an example that doesn't involve food:


//Pseudo-code (Fake code)
if (prefer GoogleChrome) {
use(Chrome);
}
if (prefer MozillaFirefox){
use(Firefox);
}
else{
use(SomethingElse);
}

Comparison Operators

There are quite a few comparison operators, but we're going to stick with the mainstream ones:


> (Greater than), < (Less than)
>= (Greater Than or Equal To), <= (Less Than or Equal To)
== (Equals) === (Equal to and Same Type)
!= (Does not Equal), !== (Not equal to and/or different type)
if(2 < 3) {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is correct".

if(3 < 2) {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is incorrect".
 if(2 != 3) {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is correct".

if(3 !== 2) {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is correct".
 if("3" == 3) {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is correct". They are only comparing value. The string is being "coerced" into a number, so their value is the same.

if(3 === "3") {
console.log('That comparison is correct');
}
else{
console.log('That comparison is incorrect');
}
//Would output "That comparison is incorrect". Because they are comparing both type and value. The string is not "coerced" into a number.
